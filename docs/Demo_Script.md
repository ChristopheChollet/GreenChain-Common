# GreenChain Common - Script de demo

## Objectif

Présenter GreenChain Common en 2 à 4 minutes en montrant un parcours utilisateur complet reliant les quatre modules du projet.

## Message d'introduction

GreenChain Common est une dApp flagship web3 climate/energy qui unifie un vault, un market, un registry et une gouvernance dans une seule experience. L'objectif est de montrer un parcours produit coherent, une UX transactionnelle simple et des KPI on-chain consolides.

## Fil conducteur

Le fil rouge de la demo est le suivant :

`Vault -> Market -> Registry -> DAO`

Autrement dit :

- le capital entre dans le systeme via le vault,
- l'activite energie/flexibilite passe par le market,
- l'impact est trace via le registry,
- les regles peuvent ensuite etre pilotees via la gouvernance.

## Script de demonstration

1. **Connexion wallet**
   - Ouvrir l'application.
   - Connecter le wallet sur le reseau Hardhat local.
   - Montrer que le dashboard et les modules sont disponibles depuis une seule navigation.

2. **Vault**
   - Aller sur `Vault`.
   - Lancer `Approve`, puis `Deposit`.
   - Expliquer que cette etape represente l'entree du capital dans le systeme.

3. **Market**
   - Aller sur `Market`.
   - Creer un slot, puis matcher ce slot.
   - Expliquer que cette brique represente une logique de flexibilite energetique integree a l'application.

4. **Registry**
   - Aller sur `Registry`.
   - Emettre un REC, puis en retirer un.
   - Expliquer que l'activite peut etre tracee via des certificats environnementaux.

5. **Governance**
   - Aller sur `Governance`.
   - Creer une proposition, puis voter.
   - Expliquer que la gouvernance permet de piloter certaines decisions de l'ecosysteme.

6. **Retour dashboard**
   - Revenir sur le dashboard.
   - Montrer le flow visuel des 4 modules.
   - Montrer l'evolution des KPI et de l'activite recente.
   - Conclure en rappelant que la valeur du projet est l'unification du parcours utilisateur et des donnees on-chain.

## Ce que la demo doit prouver

- Capacite a integrer plusieurs smart contracts dans une seule interface.
- Capacite a construire une UX transactionnelle simple et demonstrable.
- Capacite a lire des events on-chain et a les transformer en KPI.
- Capacite a relier plusieurs briques produit dans un meme flow.
